# Audit snapshots

One-shot review artifacts (file inventory, skill review, etc.). They are dated
snapshots of the skill at a point in time, **not** living documentation.

`SKILL.md` does not link here — agents follow it should never end up in this
folder. Reference docs live in `../`.

When a snapshot's findings are addressed in code or in `../`, leave the
snapshot in place as historical record; do not edit it.
